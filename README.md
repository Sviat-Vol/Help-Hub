# Help-Hub (Volontire Map)

Help-Hub — це вебплатформа взаємодопомоги у форматі мапи запитів.
Ідея Volontire Map: людина, якій потрібна допомога, створює запит із коротким описом, деталями та геолокацією, а інші користувачі бачать цей запит на мапі й можуть взяти його в роботу.

## Для кого сайт
- Для людей, яким потрібна швидка допомога поруч.
- Для волонтерів і небайдужих користувачів, які можуть допомогти на місці.

## Як користуватися (за 1 хвилину)
1. Зареєструйтесь або увійдіть у акаунт.
2. Якщо вам потрібна допомога, створіть запит на сторінці `/create_request`.
3. Додайте коротку назву, опис і геолокацію.
4. Відкрийте `/map`, щоб переглядати активні запити.
5. Натисніть `Прийняти`, щоб взяти запит у роботу.
6. Для прийнятого запиту відкрийте контакти учасників.

## Ролі користувачів
- Автор запиту: створює запит, бачить його на мапі, може видалити власний запит.
- Волонтер/виконавець: переглядає запити, приймає доступний запит, може скасувати прийняття і повернути запит у доступні.

## Правила доступу
- Створювати, приймати, скасовувати запити та переглядати контакти можуть лише авторизовані користувачі.
- Прийняти можна лише запит зі статусом `New`.
- Скасування працює так:
  - автор скасовує свій запит -> запит видаляється;
  - виконавець скасовує прийняття -> запит повертається у `New`.
- Контакти доступні тільки для учасників запиту (автор + виконавець) і тільки коли запит у статусі `Accepted`.

## Стани запиту
| Статус | Що означає |
| --- | --- |
| `New` | Запит створено, він доступний для прийняття іншим користувачем. |
| `Accepted` | Запит взято у роботу конкретним виконавцем. |

## Як працює сайт
- Backend: FastAPI + SQLAlchemy.
- Frontend: HTML/CSS/JavaScript.
- Мапа: Leaflet + OpenStreetMap.
- Дані користувачів зберігаються в `users.db`.
- Дані запитів зберігаються в `requests.db`.
- Мапа оновлюється періодично і підтягує нові/оновлені запити через API.

## Ключові сторінки
- `/` — головна сторінка.
- `/register` — реєстрація.
- `/login` — вхід.
- `/create_request` — створення запиту.
- `/map` — мапа запитів.

## Типові ситуації
- Не вдається увійти: перевірте email/пароль і чи існує акаунт.
- Геолокація не підтягується: дозвольте доступ до локації у браузері.
- Кнопка контактів не працює: контакти доступні лише для `Accepted` запиту та тільки його учасникам.

## Ключові API
- `GET /api/requests` — список запитів.
- `POST /api/requests` — створення нового запиту.
- `POST /api/requests/{request_id}/accept` — прийняття запиту.
- `POST /api/requests/{request_id}/cancel` — скасування/повернення запиту.
- `GET /api/requests/{request_id}/contacts` — контакти учасників прийнятого запиту.

## Обмеження поточної версії
- Немає push-сповіщень про нові запити.
- Немає вбудованого чату між учасниками.
- Немає рейтингу/верифікації волонтерів.
- Проєкт орієнтований на веб-версію.
