# Help-Hub (Volontire Map)

Help-Hub — це вебплатформа взаємодопомоги у форматі мапи запитів.
Ідея Volontire Map: людина, якій потрібна допомога, створює запит із коротким описом, деталями та геолокацією, а інші користувачі бачать цей запит на мапі й можуть взяти його в роботу.

## Основна концепція
- Користувачі реєструються та входять у систему.
- Авторизований користувач створює запит на допомогу.
- Запит відображається на інтерактивній мапі.
- Інший користувач може прийняти запит.
- Автор або виконавець може скасувати запит за визначеними правилами.
- Для прийнятого запиту доступний обмін контактами між учасниками.

## Ролі користувачів
- Автор запиту:
  - створює запит;
  - бачить запит на мапі;
  - може видалити власний запит.
- Волонтер/виконавець:
  - переглядає запити на мапі;
  - приймає доступний запит;
  - може скасувати прийняття й повернути запит у стан доступного.

## Як працює життєвий цикл запиту
1. Створення: новий запит отримує статус `New`.
2. Прийняття: після натискання «Прийняти» статус змінюється на `Accepted`.
3. Скасування:
   - якщо скасовує автор — запит видаляється;
   - якщо скасовує виконавець — запит знову стає `New`.

## Склад проєкту
- Backend: FastAPI + SQLAlchemy.
- Frontend: HTML/CSS/JavaScript.
- Мапа: Leaflet + OpenStreetMap.
- Зберігання:
  - користувачі в `users.db`;
  - запити на допомогу в `requests.db`.

## Ключові сторінки
- `/` — головна сторінка.
- `/register` — реєстрація.
- `/login` — вхід.
- `/create_request` — створення запиту.
- `/map` — мапа запитів.

## Ключові API
- `GET /api/requests` — список запитів.
- `POST /api/requests` — створення нового запиту.
- `POST /api/requests/{request_id}/accept` — прийняття запиту.
- `POST /api/requests/{request_id}/cancel` — скасування/повернення запиту.
- `GET /api/requests/{request_id}/contacts` — контакти учасників прийнятого запиту.
